Tab 四边拉伸设计

目标：让每个 Tab 都像一个完整面板，可从上下左右边界拖拽，但不会把标题栏、输入框或底部对话区挤坏。

A

只保留分隔条

继续只允许拖容器分隔条，不给每个 Tab 暴露四边热区。

B

每个 Tab 四边都可拖

Tab 的上下左右边界都变成热区，但实际仍映射到最近的相邻 divider。推荐。

C

悬浮控制柄

选中 Tab 后出现四边控制柄，再拖动控制柄调整尺寸。

推荐交互语义

优点

  • 用户心智自然：拖的是 Tab 自己，不是抽象布局树。
  • 实现成本可控：命中层在 Pane，实际求解仍复用现有 container solver。
  • 容易保证完整性：只允许把相邻 pane 压到最小安全尺寸，不跨层级传压。

代价

  • 需要为每个 Pane 计算四边可拖性和映射到哪条 divider。
  • 边角区域要处理优先级，避免同时命中两个方向。

完整性保护规则

1. 标题栏永远完整

任何 Tab 都不能被压到标题栏不可见。

2. 底边对话框优先保留

触底的 dialog pane 需要完整 composer 和最小内容区。

3. 只影响相邻面板

拖某个边界只改与该边相邻的一对 pane，不继续向更远 pane 传压。
